# Architecture — Initial Direction

## Purpose

The prototype will review a candidate change in a local e-commerce repository, produce evidence-based feedback, propose but not automatically merge a correction, validate that correction in an isolated Git worktree, and emit a policy-driven readiness decision.

## Initial architecture boundary

The design deliberately separates deterministic operations from language-model reasoning.

```text
Git candidate change
  ├─ deterministic context collection
  ├─ deterministic checks: diff integrity, tests, lint, build
  ├─ bounded local-model analysis
  ├─ evidence validation against repository files
  ├─ temporary-worktree patch validation
  └─ policy-driven readiness decision
```

## Components planned

| Component | Responsibility | Trust level |
|---|---|---|
| CLI | Starts a review run and exposes explicit inputs. | Trusted orchestration |
| Git tools | Collect diffs, create temporary worktrees, apply patches only outside the source branch. | Deterministic |
| Check runner | Executes allowlisted commands and captures exit codes/output. | Deterministic |
| Context builder | Selects bounded relevant code, tests, docs, and check results. | Deterministic |
| Local-model client | Requests structured review or patch proposals. | Advisory only |
| Evidence validator | Confirms claimed file/line/evidence exists. | Deterministic |
| Policy engine | Converts objective checks and validated findings into a readiness result. | Deterministic |
| Artifact writer | Creates reproducible Markdown and JSON outputs. | Deterministic |

## Non-goals for the first delivery

- Autonomous commits, merges, or pushes.
- Arbitrary shell execution generated by a model.
- Full repository semantic indexing or vector databases.
- A web user interface.
- Cloud-only model dependency.
- Broad multi-agent orchestration for its own sake.

## Why this architecture

The challenge prioritizes a focused prototype. This design concentrates complexity where it creates demonstrable value: evidence, isolated validation, explicit decision criteria, reproducibility, and privacy.
