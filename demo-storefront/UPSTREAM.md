## Why this project was selected

It is intentionally small enough to be understood, tested, and modified within the challenge timebox while still providing an e-commerce domain context for cart, checkout, shipping, pricing, or inventory-related review scenarios.

## Repository-history handling

The upstream `.git` directory was intentionally removed before this directory became part of the prototype repository. This lets the final deliverable use one cohesive Git history while retaining upstream attribution and licensing.

## Controlled modifications for the technical challenge

Changes introduced in this directory after the baseline audit are challenge-specific. Expected changes include:

- Checkout business-rule documentation.
- Tests for selected checkout behavior.
- Controlled candidate changes that intentionally introduce a realistic regression.
- Supporting files required to run deterministic validation.

A defect introduced for a controlled scenario must never be represented as a defect found in the upstream project unless separate evidence establishes that fact.
